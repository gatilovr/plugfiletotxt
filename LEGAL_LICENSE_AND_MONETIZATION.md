# Legal, License, and Monetization Guidance

## Recommended License for Your Case

Given your current model (free plugin + support/donations now, possible monetization later), the best default is:

- **Apache-2.0** (recommended)

Why Apache-2.0:

- Permissive for users and contributors.
- Commercial use allowed.
- Better explicit patent terms than MIT.
- Safe path if you later add paid extensions/freemium model.

MIT is also valid, but Apache-2.0 is usually stronger for future commercial flexibility.

## Marketplace Commercial Strategy

Phase 1 (now):

Phase 2 (later):

- Alternative: release a separate paid “Pro” plugin with enhanced workflows.

## Minimum Legal Package Before Listing

1. `LICENSE` file in repository (done).
2. `NOTICE` file (done).
3. Short privacy statement (recommended for listing and README):
   - plugin runs locally,
   - no external data transfer by default,
   - user selects source/output folders manually.
4. Support policy:
   - where users report issues,
   - expected response window.
5. Disclaimer:
   - user is responsible for exported content and local storage destination.
